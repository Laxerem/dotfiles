# EWW Panel Configuration - Шпаргалка для Claude

## Обзор проекта

Кастомная панель статуса для Linux с использованием EWW (ElKowar's Wacky Widgets).

**Путь к конфигурации:** `/home/laxerem/.config/my_eww/`

**ВАЖНО:** Пользователь использует нестандартный путь! Запускать команды нужно с флагом `-c`:
```bash
eww -c /home/laxerem/.config/my_eww/ <command>
```

## КРИТИЧЕСКИ ВАЖНО: Проблема автоматических отступов

### Проблема
В EWW элементы внутри `box` автоматически растягиваются и получают динамические отступы в зависимости от длины содержимого. Изменение CSS (padding, margin) **НЕ ПОМОГАЕТ** решить эту проблему!

### Симптомы
- Отступы между элементами меняются в зависимости от длины текста
- При разной длине контента элементы "прыгают"
- CSS правила `padding`, `margin`, `spacing` не влияют на отступы

### Решение: Фиксированная структура в eww.yuck

**НЕПРАВИЛЬНО:**
```yuck
(box :orientation "horizontal" :spacing 10
  (label :text "${VARIABLE_TEXT}")
  (image :path "icon.svg")
)
```

**ПРАВИЛЬНО:**
```yuck
(box :orientation "horizontal" :space-evenly false
  (box :hexpand false
    (label :text "${VARIABLE_TEXT}")
  )
  (box :hexpand false
    (image :path "icon.svg")
  )
)
```

### Ключевые параметры

1. **`:space-evenly false`** - на родительском box
   - Отключает равномерное распределение пространства

2. **`:hexpand false`** - на каждом дочернем box
   - Запрещает горизонтальное растяжение элемента

3. **`:vexpand false`** - для вертикального направления
   - Запрещает вертикальное растяжение элемента

### После фиксации структуры

Только после применения этих параметров CSS margin/padding начинают работать корректно:

```scss
.element-icon {
  margin-right: 8px;  // Теперь это даст фиксированный отступ 8px
}
```

## Особенности работы с устройствами

### Определение наушников
Проверка в defpoll HEADPHONES:
```yuck
(defpoll HEADPHONES :interval "2s"
  "pactl get-default-sink | grep -q 'bluez_output\\|headphone\\|headset' && echo 'true' || echo 'false'")
```

### Иконки аудио устройств
- `󰕾` - Динамики (обычный режим)
- `󰖁` - Динамики (muted, красный)
- `󰋋` - Наушники (обычный режим)
- `󰟎` - Наушники (muted, красный)

### Генерация списка устройств

Скрипт `gen_audio_sinks.sh` генерирует yuck-код динамически. Важно:
- Возвращать всё в **одной строке** (deflisten читает построчно)
- Обернуть всё в общий box
- Использовать структуру с :hexpand false для каждого элемента

## Анимация зарядки батареи

### Статусы зарядки
- `Charging` - активная зарядка
- `Not charging` - подключено к питанию, но зарядка приостановлена (>80%)
- `Discharging` - работа от батареи

**ВАЖНО:** При заряде >80% система часто показывает "Not charging". Условие должно проверять **оба** статуса:

```yuck
:class "${CHARGE_STATUS == 'Charging' || CHARGE_STATUS == 'Not charging' ? ...}"
```

### Классы анимации
- `bat-charging-low` - ≤30%, красная анимация, 3s
- `bat-charging-medium` - ≤80%, желтая анимация, 15s
- `bat-charging-high` - >80%, зеленая анимация, 15s (или 3s для лучшей видимости)

## КРИТИЧЕСКИ ВАЖНО: Всплывающие окна (popup) — правила открытия/закрытия

### Правильный паттерн — ПРОВЕРЕН, НЕ МЕНЯТЬ

**В баре** (eventbox на виджете):
```yuck
:onhover "eww -c ... update hover_var=true; ~/.config/.../open_popup.sh popup-name"
:onhoverlost "eww -c ... update hover_var=false"
```
⚠️ **НЕ вызывать close и НЕ вызывать close_popup_delayed.sh из onhoverlost бара!** Только сброс переменной.

**В самом popup** (внешний eventbox):
```yuck
:onhover "eww -c ... update hover_var=true"
:onhoverlost "eww -c ... update hover_var=false; eww -c ... close popup-name"
```
Закрытие — **только отсюда**, прямое, без задержек и скриптов.

**Параметры окна:**
```yuck
(defwindow my-popup
  :stacking "overlay"
  :windowtype "normal"
  :wm-ignore true
  ; :focusable true — ТОЛЬКО если нужен input для ввода текста
```

### Почему именно так — разбор граблей

**Грабля 1: Закрытие из `onhoverlost` бара вызывает мигание (flicker)**

Когда popup с `:focusable true` открывается, он перехватывает pointer events. Бар получает `leave` → запускает закрытие. Popup закрывается → бар получает `enter` → открывает снова. Бесконечный мигающий цикл. **Решение: бар вообще не закрывает popup.**

**Грабля 2: Сложные скрипты с `hyprctl cursorpos` и задержками — ненадёжны**

Race condition: задержка истекает до того, как popup успевает зарегистрировать hover. Popup закрывается когда не должен. **Решение: прямое `eww close` в onhoverlost самого popup, без скриптов и задержек.**

**Грабля 3: `:focusable true` блокирует экран — НЕ самостоятельная проблема**

Казалось что `:focusable true` блокирует всё. На самом деле это было следствием грабли 2: popup получал фокус + не закрывался → тупик. С правильной close-логикой `:focusable true` работает нормально и нужен для полей ввода.

**Грабля 4: popup и бар перекрываются по координатам (оба y=0)**

Это нормально и даже помогает: когда курсор "уходит" с бара в popup — он фактически всё ещё внутри popup. Popup закрывается только когда курсор реально покидает его область.

### Аварийные кейбинды (на случай если что-то пойдёт не так)
- `Super + F12` — закрыть wifi-popup аварийно
- `Super + Shift + F12` — убить весь eww и перезапустить бар

Эти биндинги перехватываются Hyprland на уровне compositor, работают даже если какое-то окно захватило весь ввод.

## Траблшутинг

### Проблема: Отступы не изменяются при изменении CSS

**Решение:** Проблема не в CSS! Нужно изменять структуру в eww.yuck:
1. Добавить `:space-evenly false` на родительский box
2. Обернуть каждый элемент в box с `:hexpand false`
3. Только после этого CSS margin/padding будут работать

### Проблема: Элементы "прыгают" при изменении текста

**Решение:** Использовать фиксированную структуру (см. выше) или `:limit-width` для label:
```yuck
(label :text "${TEXT}" :limit-width 15 :truncate true)
```

### Проблема: deflisten показывает только последнюю строку

**Причина:** deflisten читает stdout построчно и каждая новая строка перезаписывает предыдущую.

**Решение:** Генерировать всё в одной строке:
```bash
echo -n "(box :spacing 5 "
# ... генерация элементов ...
echo ")"
```

### Проблема: literal widget не отображает содержимое

**Возможные причины:**
1. Неправильное экранирование кавычек в скрипте
2. deflisten возвращает несколько строк (см. выше)
3. Синтаксическая ошибка в сгенерированном yuck-коде

**Решение:** Проверить вывод скрипта вручную и убедиться, что это валидный yuck-код в одной строке.

### Проблема: Вертикальные отступы между блоками не изменяются

**Корневая причина:** По умолчанию `space-evenly` равно `true` в EWW. Это значит,
что дочерние элементы box делят пространство контейнера **поровну**, независимо от
размера содержимого. Никакие CSS правила (margin, padding) это НЕ исправят.

**Решение:** Добавить `:space-evenly false` на **каждый** вертикальный box-контейнер,
включая самый внешний:
```yuck
(box :orientation "vertical" :space-evenly false
  (box :vexpand false ...)
  (box :vexpand false ...)
)
```

**ВАЖНО:** Если ранее использовались отрицательные `margin-top` как костыль,
их нужно УБРАТЬ после добавления `:space-evenly false` — иначе блоки будут
накладываться друг на друга.

## Wi-Fi Popup

### Открытие
Всплывающее окно открывается при наведении курсора на название Wi-Fi сети или иконку.

### Сортировка сетей
**ВАЖНО:** Сети отсортированы по приоритету:
1. **Сначала**: Сохраненные сети (к которым ранее подключались), отсортированные по силе сигнала
2. **Затем**: Несохраненные сети, отсортированные по силе сигнала

### Визуальные отличия

#### Сохраненные сети:
- ✅ Зеленая иконка сигнала (цвет #a6e3a1)
- ✅ Без иконки замка
- ✅ Подключение в один клик без ввода пароля

#### Несохраненные сети:
- 🔵 Обычная иконка сигнала (белая)
- 🔒 Иконка замка 󰌾 для защищенных сетей
- 🔑 Требуют ввод пароля для подключения

#### Текущая подключенная сеть:
- Выделена синим фоном независимо от статуса сохраненности

### Функциональность
- Автоматическое удаление дубликатов (показывается только точка с лучшим сигналом)
- Иконки уровня сигнала: 󰤨 (отличный ≥75%), 󰤥 (хороший ≥50%), 󰤢 (средний ≥25%), 󰤟 (слабый <25%)
- Кнопка "󰑓 Обновить" для повторного сканирования

### Подключение к сети

#### Сохраненная сеть (зеленая):
1. Клик по сети → автоматическое подключение
2. Уведомление об успехе/ошибке

#### Новая сеть (белая с замком):
1. Клик по сети → открывается секция ввода пароля внутри popup (revealer)
2. Ввод пароля
3. Enter или кнопка "Подключить" → подключение с сохранением
4. Уведомление об успехе/ошибке

**Про `:focusable true` и блокировку экрана:**
Изначально казалось, что `:focusable true` на wifi-popup блокирует весь экран. На самом деле блокировка была **следствием бага с незакрытием**: popup получал фокус, не закрывался, и пользователь не мог ничего сделать. После исправления close-логики (прямое закрытие в `onhoverlost` без сложных скриптов), `:focusable true` работает нормально — нужен для поля ввода пароля.

### Переменные EWW
- `wifi_hover` - курсор над wifi-виджетом или popup
- `wifi_show_password` - показывать ли секцию ввода пароля
- `wifi_connect_ssid` - SSID сети для подключения
- `wifi_password` - введенный пароль (временное хранение)

### Зависимости
- `nmcli` - для управления сетями (NetworkManager)
- `notify-send` - для уведомлений
