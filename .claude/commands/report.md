Сгенерируй отчёт о прогрессе и отправь его в Redmine.

## Шаг 1 — Разбери аргументы

Аргументы: `$ARGUMENTS`

Парсь так:
- Первый аргумент — номер задачи Redmine (обязательный)
- `--day` — взять все коммиты за сегодня (с midnight)
- `--N` где N — число — взять последние N коммитов (напр. `--2`, `--5`)
- `--auto` — отправить сразу без подтверждения
- Если ни флага нет — по умолчанию `--1` (последний коммит)

Примеры:
- `/report 42` → последний коммит, показать превью, ждать подтверждения
- `/report 42 --day` → все коммиты за сегодня, показать превью, ждать подтверждения
- `/report 42 --3` → последние 3 коммита, показать превью, ждать подтверждения
- `/report 42 --day --auto` → все коммиты за сегодня, отправить сразу

## Шаг 2 — Получи данные коммитов

Если `--day`:
```bash
git log --since="midnight" --format="%H|%s|%ad" --date=format:"%d.%m.%y"
git diff $(git log --since="midnight" --format="%H" | tail -1)^ HEAD --stat
git diff $(git log --since="midnight" --format="%H" | tail -1)^ HEAD
```

Если `--N` (например --3):
```bash
git log -3 --format="%H|%s|%ad" --date=format:"%d.%m.%y"
git diff HEAD~3 HEAD --stat
git diff HEAD~3 HEAD
```

Если без флага (по умолчанию --1):
```bash
git log -1 --format="%H|%s|%ad" --date=format:"%d.%m.%y"
git diff HEAD~1 HEAD --stat
git diff HEAD~1 HEAD
```

Также получи remote URL для ссылки на коммит:
```bash
git remote get-url origin
git rev-parse --short HEAD
git rev-parse HEAD
git branch --show-current
```

## Шаг 3 — Анализируй изменения

На основе diff определи:
- Что было сделано (группируй по смыслу, не по файлам)
- Какой результат получился — что теперь работает / появилось
- Не углубляйся в детали реализации — уровень фич и архитектурных слоёв

## Шаг 4 — Составь отчёт в формате Textile

Формат строго такой:

```
h3. Прогресс дня (ДД.ММ.ГГ)
1. *Что сделано:*
* Подпункт
2. *Что сделано:*
* Подпункт
Результат выполненной работы, и более подробные изменения в +последнем комите+ :
*ветка* -> *короткий_хеш*
https://github.com/OWNER/REPO/commit/ПОЛНЫЙ_ХЕШ
```

Правила:
- Дата — из последнего коммита в выборке
- Пункты = логические группы изменений
- Каждый пункт начинается с жирного глагола (*Реализовал*, *Добавил*, *Разбил* и т.д.)
- Ссылку на коммит строй из remote origin URL + полный хеш последнего коммита
- Если коммитов несколько — в конце всё равно ссылка на последний

## Шаг 5 — Превью и отправка

Всегда сначала выведи отчёт пользователю в виде превью:

```
📋 Превью отчёта для задачи #ISSUE_ID:
─────────────────────────────────────
<сгенерированный отчёт>
─────────────────────────────────────
```

Если передан флаг `--auto`:
- Сразу отправляй без вопросов, выведи "✅ Отправлено автоматически"

Если `--auto` НЕТ:
- После превью спроси: "Отправить отчёт? (да / нет / редактировать)"
- `да` — отправляй
- `нет` — отмена, ничего не отправляй
- `редактировать` — попроси написать что изменить, исправь отчёт и покажи превью снова

Команда для отправки:
```bash
curl -s -X PUT "$REDMINE_URL/issues/$ISSUE_ID.json" \
  -H "X-Redmine-API-Key: $REDMINE_API_KEY" \
  -H "Content-Type: application/json" \
  -d "{\"issue\": {\"notes\": \"$REPORT_TEXT\"}}"
```

После успешной отправки выведи ссылку на задачу: `$REDMINE_URL/issues/$ISSUE_ID`
